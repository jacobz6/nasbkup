## 1. 产品概述

NAS Backup 是一个 NAS 文件冷备份管理系统，将 NAS 上的文件备份到阿里云 OSS 冷存储，支持增量备份、内容去重、压缩、端到端加密、实时进度反馈、数据一致性对账等核心功能。前端为管理控制台，面向 NAS 管理员，提供备份状态监控、内容配置、策略管理、数据对账和日志审计能力。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 管理员 | 直接访问 | 全部功能：监控、配置、触发备份、数据对账、查看日志 |

### 2.2 功能模块

1. **全览页面**：备份状态概览（含SSE实时进度条）、容量仪表盘、快捷统计卡片、操作按钮、备份历史、对账告警
2. **内容选择页面**：备份目录管理、文件大小限制、排除规则管理、目录部分备份状态标识
3. **策略设置页面**：定时任务配置、压缩级别、上传设置（含并发数）、保留策略、加密配置、删除策略
4. **日志页面**：备份日志查看、级别过滤、关键词搜索、时间范围筛选
5. **数据对账页面**：数据一致性检查与修复（DryRun模式、自动修复、问题分类展示）、OSS健康检查

## 3. 核心流程

### 3.1 备份触发流程（SSE实时进度）

用户在全览页面点击"一键备份"/"全量备份"按钮 → 前端调用 POST /api/backup/trigger（type=auto/full/incremental）→ 后端返回 202（backupID和pending状态）→ 前端通过 SSE (EventSource) 订阅 /api/backup/progress/stream 接收实时事件 → 显示阶段切换（扫描→哈希→去重→上传→完成）、百分比进度条、当前处理文件名、实时日志 → 收到 completed/failed/cancelled 事件后停止进度显示 → 刷新仪表盘和历史数据

**备份中**：SSE推送为主，不依赖轮询；SSE断开时自动降级为2秒轮询。
**空闲时**：每10秒轮询仪表盘数据。

### 3.2 智能备份类型判断（auto）

用户点击"一键备份"时传 type=auto → 后端 determineBackupType() 判断：
- 如果没有成功完成的全量备份 → 执行全量备份
- 如果最近全量备份超过 full_reset_interval 月 → 执行全量备份
- 否则 → 执行增量备份

### 3.3 崩溃恢复流程

应用启动时：
1. 执行 CleanupStaleRunning()，将所有残留的 pending/running 状态备份标记为 failed
2. 初始化 ProgressBroker、Engine、各模块
3. 启动 HTTP 服务和调度器

用户点击"取消备份"时：
- 首先在内存中查找运行中的备份并取消
- 如果内存中找不到（崩溃场景），检查DB中是否有running/pending记录，清理为failed

### 3.4 数据一致性对账流程

用户在对账页面点击"DryRun检查" → 前端调用 POST /api/reconcile?dry_run=true → 后端获取备份锁（与备份互斥，运行中返回409）→ 交叉比对 OSS对象 ↔ hash_index ↔ backup_files ↔ backup状态 → 返回 ReconcileReport（列出各类不一致，不修改数据）→ 前端分类展示问题。

用户点击"执行修复"（二次确认后）→ POST /api/reconcile?dry_run=false → 后端执行相同检查并自动修复可修复项（悬空索引删除、ref_count修正、孤立backup_files清理、状态修正），OSS孤儿对象仅记录不删除 → 返回修复报告。

### 3.5 配置修改流程

用户在策略页面修改配置 → 前端调用 PUT/PATCH /api/strategy/* 或 /api/content/* → 后端更新数据库 config_kv 表并同步运行时配置 → 返回更新后的配置 → 前端显示保存成功Toast

### 3.6 日志查询流程

用户在日志页面选择过滤条件 → 前端调用 GET /api/logs（带查询参数）→ 后端返回分页日志列表 → 前端渲染日志表格 → 用户翻页或修改筛选条件时重新请求

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深蓝灰（#1e293b）为底，冰蓝（#38bdf8）为强调色，搭配暗色系仪表盘风格
- **辅助色**：翠绿（#34d399）表示成功/完成，琥珀（#fbbf24）表示警告，玫红（#f43f5e）表示错误
- **按钮风格**：圆角（8px），主要按钮带微妙渐变，次要按钮为描边风格
- **字体**：标题使用 JetBrains Mono（等宽技术感），正文使用 Noto Sans SC（中文优化）
- **布局风格**：左侧固定导航栏（5个导航项）+ 右侧内容区，卡片式布局，深色主题
- **图标风格**：使用 Lucide 图标库，线条风格，2px 描边

### 4.3 响应式设计

- 桌面优先设计，最小宽度 1024px
- 导航栏在窄屏下折叠为图标模式
- 统计卡片在中等屏幕下从4列变为2列
- 表格在窄屏下支持横向滚动

### 4.4 动效设计

- 页面切换：淡入淡出（200ms）
- 卡片加载：骨架屏占位
- 备份运行中：状态指示器脉冲动画 + 进度条平滑增长
- SSE事件到达：数值/进度平滑过渡
- 按钮交互：hover 上浮 + 阴影加深
- 数据更新：数值变化时平滑过渡
- Toast通知：从右上角滑入，4秒后自动消失
